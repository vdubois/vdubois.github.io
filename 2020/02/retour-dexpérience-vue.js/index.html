Retour d'expérience Vue.JS

@Vincent Dubois · Feb 15, 2020 · 4 min. de lecture

Après un an d’utilisation de Vue.JS en entreprise, voici un petit retour d’expérience dans lequel je vais vous partager mon ressenti sur l’utilisation de ce framework dans le cadre d’un vrai projet.

Le premier atout que l’on découvre assez vite quand on se lance avec Vue.JS, c’est sa simplicité. Pas de fioritures, juste l’essentiel permettant de réaliser des développements professionnels.

La communauté Vue.JS est maintenant assez conséquente, et on trouve sans aucun problème une multitude de composants ou de plugins open source pour tous les besoins. Personnellement, j’aime bien le github “Awesome Vue”.

La société pour laquelle je travaille utilisait déjà Vue.JS, mais un peu “à l’arrache”, comme on dit. Un des gros avantages de Vue.JS, c’est que vous pouvez l’intégrer très facilement dans n’importe quel site internet ou application web. Je me suis alors attaché à intégrer Vue.JS dans notre projet de la façon la plus classique et industrialisée qui soit (en utilisant vue-cli notamment).

Intégrer Vue dans de l’existant

Intégrer Vue dans de l’existant est très simple !

Et pourtant, le framework maison de ma société utilise Require.JS pour le chargement des ressources. Il m’a suffi d’ancrer l’instance de Vue sur une balise d’une page existante (donc exactement comme fonctionne Vue de base).

if (document.getElementById('monDiv')) {
	new Vue({
	  render: (h: any) => h(MonComposant)
	}).$mount('#monDiv')
}

Ensuite, lorsque la page est chargée par le framework maison, je déclenche le démarrage de l’application.

<div>
    <div id="monDiv">
    </div>
	
	<script src="../webapp/spa/js/app.js"></script>
    <script src="../webapp/spa/js/chunk-vendors.js"></script>
</div>

Du coup, ce n’est pas une single page application à proprement parler, mais ça n’empêche pas qu’elle le soit en mode développeur, et de faire un bundle pour l’application finale comme ci-dessus.

Single File Components

La syntaxe de composants Vue peut être réalisée de plusieurs manières différentes. J’ai l’impression que dans la communauté Vue, la voie privilégiée est de faire des “Single File Components”.

Cela veut dire que l’on déclare un composant et tous ses artefacts dans un seul fichier d’extension .vue :

<template>
  <div>{{value}}</div>
</template>

<script lang="ts">
  export default {
    name: 'my-component',
    data() {
      return {
        value: 'texte'
      }
    }
  }
</script>

<style scoped>
</style>

Personnellement, je n’aime pas trop cette syntaxe. Je trouve que cela mêle un peu trop les responsabilités.

Sur notre projet, on déclare les composants avec la syntaxe Vue.component :

import * as template from './my-component.html';
import './my-component.css';

Vue.component('my-component', {
  data() {
    return {
      value: 'texte'
    }
  }
});

Le flux de données de Vue.JS

Comme Angular et React, Vue utilise la notion de composants parents et enfants.

Un composant parent va faire transiter les données vers ses composants enfants au travers de leurs ‘props’ (attributs des tags des composants).

Injection de dépendances

Vue possède une fonctionnalité d’injection de dépendances native.

On déclare tout d’abord un provider de dépendances :

export const ServicesProvider = {
  monService: new MonService()
}

Ensuite, on fournit ce provider dans le composant de plus haut niveau :

import { ServicesProvider } from '@/services/services-provider';

Vue.component('mon-composant-principal', {
  provide: ServicesProvider
})

Cela permet ensuite de pouvoir injecter toutes les dépendances dans les composants enfants :

import { ServicesProvider } from '@/services/services-provider';

Vue.component('mon-composant', {
  inject: ['monService'],
  methods: {
    methode() {
      this.monService.methodeDeService();
    }
  }
})

Comme on peut le voir, quand on déclare puis on injecte une dépendance ‘monService’, un nouveau champ ‘monService’ est ajouté à l’instance du composant Vue.

Le seul défaut de ce système est que l’on ne peut pas injecter de dépendances sur le composant de plus haut niveau, étant donné que c’est le composant qui fournit ces dépendances.

Mixins

Vue fournit le mécanisme de mixin, qui permet d’ajouter des mêmes fonctionnalités à plusieurs composants. Cela va fusionner le code du mixin au code du composant sur lequel il est appliqué.
